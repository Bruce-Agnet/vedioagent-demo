# 管理后台 · 剩余板块 PRD

> 本文档只描述**产品需求与逻辑**（"做什么 / 为什么 / 在什么状态间流转"），不涉及技术实现（数据结构、组件、样式）。研发会另出技术文档。

---

## 0. 背景

一期已上线 4 个 P0 模块：
- 数据看板（DAU / MAU / 营收 / Top 项目 / 模型分布）
- 用户管理（搜索 / 封禁 / 赠送积分）
- 订单与计费（订单查询 / 退款）
- 模型与计价（模型 CRUD / 上下架）

但前台已具备的产品体系（项目 + 会话 + 作品 + 成员 + 配额 + 技能 + 提示词）只覆盖了「用户」和「订单」两条线，剩余 4 条主线尚未在 admin 侧落地：

1. **内容治理** — 项目 / 作品 / 技能 / 提示词 / 公告
2. **营收营销** — 套餐 / 优惠券 / 财务对账
3. **权限安全** — 多管理员账号 / 角色 / 操作审计
4. **客服反馈** — 工单 / 用户反馈

本 PRD 覆盖以上全部 11 个子模块。

---

## 2. M1 · 项目管理

### 价值
运营需对全平台项目做盘点、隔离违规项目、追查高消耗项目。前台 owner 视角无法跨用户看项目。

### 用户故事
- 收到投诉 → 搜索项目 → 查看项目内的作品 → 决定隔离/封禁
- 月度盘点 → 找出 Top 50 消耗项目 → 评估盈亏
- 临时排障 → 进入指定项目查看创作活动

### 功能清单
1. **项目列表**
   - 搜索：项目名 / Owner 手机号
   - 筛选：状态、创建时间区间
   - 排序：累计消耗 / 创建时间 / 最近活跃
   - 列：项目名 / Owner / 成员数 / 作品数（视频/图片/文档）/ 累计预算 / 累计消耗 / 最近活跃 / 状态
2. **项目详情抽屉**
   - 基本信息（名称 / Owner / 创建时间）
   - 成员列表（手机号 / 角色 / 配额 / 已用）
   - 作品计数和最近创作时间线
   - 累计预算 vs 累计消耗对比
3. **行操作**
   - 隔离 / 解除隔离
   - 封禁 / 解除封禁（需二次确认）
   - 写入操作审计日志

### 项目状态
- `active`（正常）— 默认
- `isolated`（已隔离）— 前台对成员仍可见，但不可继续创作；不计入排行
- `banned`（已封禁）— 前台所有人都看不到，作品打码

### 字段
- 项目 ID / 名称 / Owner 手机号
- 创建时间 / 最近活跃
- 成员数 / 作品数 {视频, 图片, 文档}
- 累计预算 / 累计消耗
- 状态 / 风险标签（可选）

## 3. M2 · 作品审核

### 价值
合规需求：用户举报、平台主动巡检违规内容（涉黄/暴力/版权）。

### 用户故事
- 用户举报某个作品 → 运营定位 → 隐藏或删除 → 通知 Owner
- 巡检：按"近 7 天" + "图片" 浏览新作品，发现违规即处置

### 功能清单
1. **作品浏览**
   - 顶部 tab：视频 / 图片 / 文档
   - 瀑布流缩略图（视频和图片）/ 卡片列表（文档）
   - 单击放大查看
2. **筛选**
   - 项目 / 用户手机号 / 模型 / 时间范围 / 状态
3. **作品详情面板**
   - 缩略图 / 元信息（创建者 / 模型 / 参数 / 提示词摘要）
   - 引用文档（如有）
4. **处置操作**
   - 隐藏（仅 Owner 可见，不计入推荐）
   - 删除（彻底不可见）
   - 标记申诉中
   - 批量操作

### 作品状态
- `normal` — 默认
- `hidden` — 隐藏
- `banned` — 已删除
- `appeal` — 用户申诉中

### 字段
- 作品 ID / 项目 ID / 会话 ID / 创建者手机号
- 类型（image / video / doc）/ 模型 / 提示词摘要
- 创建时间 / 状态 / 处置原因 / 处置人

## 4. M3 · 技能库管理

### 价值
前台用户在文本创作场景下可勾选"剧本创作 / 剧本拆解 / 分镜脚本"等内置技能，技能本质是预置 system prompt。目前硬编码在前端，运营无法增减或调优。

### 用户故事
- 上线新技能（如"短视频文案生成"）→ 编辑 prompt → 上架
- 某技能效果差 → 暂停 / 调优 prompt 重新发布
- 看哪些技能最受欢迎 → 决定是否做精品化

### 功能清单
1. **技能列表**
   - 列：名称 / 简介 / 适用类型 / 状态 / 引用次数 / 最近更新
2. **新增 / 编辑**
   - 名称（≤16 字）
   - 简介（≤40 字）
   - 适用类型（仅文本 / 通用）
   - 内容（Markdown system prompt，无字数限制）
   - 状态（上架 / 下架）
3. **统计**
   - 最近 30 天被应用次数

### 技能状态
- `active`（上架）/ `offline`（下架）

### 字段
- 技能 ID / 名称 / 简介 / 适用类型
- system prompt 全文（Markdown）
- 状态 / 引用次数 / 最近更新

### 边界
- 技能 = 文本创作 system prompt（不含图片/视频示例）
- 图片/视频 few-shot 示例在 M4 提示词库管

## 5. M4 · 提示词库管理

### 价值
图片/视频创作时，用户可参考运营预置的提示词样例（含缩略图 + 完整提示词 + 适用模型）。需有后台维护入口。

### 用户故事
- 上新热门风格（"漫画线稿"）→ 配缩略图 + 提示词 → 上架
- 季节性活动 → 推出节日主题提示词 → 时间到自动下架

### 功能清单
1. **提示词列表**
   - 列：缩略图 / 标题 / 类型 / 标签 / 适用模型 / 引用次数 / 状态 / 最近更新
   - 筛选：类型（图片 / 视频）、标签、状态
2. **新增 / 编辑**
   - 标题（≤20 字）
   - 类型（image / video）
   - 缩略图（上传 / URL）
   - 提示词全文
   - 适用模型（多选）
   - 标签（自由输入，逗号分隔）
   - 排序权重
   - 状态
3. **批量上下架**

### 提示词状态
- `active` / `offline`

### 字段
- 提示词 ID / 标题 / 类型
- 缩略图 url / 提示词全文
- 适用模型数组 / 标签数组 / 排序权重
- 状态 / 引用次数 / 最近更新

## 6. M5 · 公告与 Banner

### 价值
触达用户的两种主要方式：公告（消息中心 / 弹窗）和 Banner（首页横幅）。

### 用户故事
- 平台维护通知 → 发全员公告
- 新功能发布 → 配首页 Banner + 跳转新功能
- 节日活动 → Banner + 公告联动

### 功能清单 · 公告
1. 列表（标题 / 投放范围 / 起止时间 / 触达数 / 状态）
2. 新增 / 编辑
   - 标题 / 正文（Markdown）
   - 投放范围：全员 / 新用户（注册 ≤ N 天）/ 指定状态 / 指定手机号列表
   - 展现方式：弹窗一次 / 消息中心常驻
   - 起止时间
3. 上下线开关
4. 统计：触达 / 阅读

### 功能清单 · Banner
1. 列表（缩略图 / 位置 / 起止时间 / 优先级 / 状态）
2. 新增 / 编辑
   - 图片（横幅尺寸建议显示）
   - 跳转 URL（站内 / 站外）
   - 位置（首页顶部 / 项目列表顶部 / 账户页顶部）
   - 起止时间 / 优先级(多个时按权重排)
3. 预览效果
4. 上下线

### 状态
- 公告：`draft` / `scheduled` / `active` / `expired`
- Banner：`active` / `offline`

## 7. M6 · 充值套餐管理

### 价值
前台充值页展示的套餐由后台配置（如"100¥=1000 积分 + 赠 100"）。一期硬编码，二期可由运营动态调整。

### 用户故事
- 上线新套餐档位 → 配置价格 / 积分 / 赠送规则
- 双 11 调价 → 临时上线"超值套餐"
- 看哪些套餐卖得好 → 据此调整主推位

### 功能清单
1. **套餐列表**
   - 列：名称 / 价格（¥）/ 基础积分 / 赠送积分 / 适用人群 / 销量 / 状态
   - 拖拽排序（决定前台展示顺序）
2. **新增 / 编辑**
   - 名称
   - 价格（人民币元）
   - 基础积分 / 赠送积分
   - 适用人群：全员 / 新人（注册 ≤ 7 天）/ 老用户
   - 限购规则：无限 / 每人 N 次 / 每日 N 次
   - 状态
3. **数据**
   - 销量 / 转化率（mock）

### 字段
- 套餐 ID / 名称 / 价格 / 基础积分 / 赠送积分
- 适用人群 / 限购规则
- 排序权重 / 销量 / 状态

## 8. M7 · 优惠券

### 价值
营销工具：促进充值转化、唤醒沉睡用户。

### 用户故事
- 大促前批量发券到所有用户
- 给沉睡用户单独发"回归礼包券"
- 看哪些券被核销了，评估活动效果

### 功能清单
1. **券模板**
   - 列：名称 / 类型 / 面额 / 有效期 / 发放总量 / 已核销 / 状态
   - 新增：
     - 名称
     - 类型：满减（满 ¥X 减 ¥Y）/ 直送积分（送 N 积分）
     - 阈值与面额
     - 有效期（绝对日期 or 领取后 N 天）
     - 发放总量上限
     - 适用套餐（全部 / 指定）
2. **发放**
   - 选择目标用户：按手机号列表 / 按筛选条件（状态、活跃度、累计消耗范围）
   - 选择券模板 + 数量
   - 立即发放 or 定时发放
3. **核销追踪**
   - 哪些用户用了，关联订单
   - 核销率统计

### 状态
- 券模板：`active` / `paused` / `expired`
- 券实例：`unused` / `used` / `expired`

### 字段
- 券模板：ID / 名称 / 类型 / 阈值 / 面额 / 有效期 / 发放总量 / 已核销 / 适用套餐 / 状态
- 券实例：ID / 模板 ID / 用户 ID / 领取时间 / 使用时间 / 关联订单 / 状态

## 9. M8 · 财务对账 & 退款审批

### 价值
财务月度对账 + 退款工单流转。一期"订单退款"是一键直接退（无审批），不适合真实场景。

### 用户故事
- 财务月底导出本月对账单
- 用户申请退款 → 运营受理 → 财务审批 → 通过/拒绝
- 月度复盘：收入 vs 各模型成本，看哪些模型亏损

### 功能清单
1. **对账单**
   - 时间维度：按日 / 周 / 月
   - 渠道维度：wechat / alipay / card
   - 列：日期 / 渠道 / 订单数 / 总金额 / 退款数 / 退款金额 / 净额
   - 导出 CSV
2. **退款审批流**
   - 工单列表（订单 ID / 用户 / 申请金额 / 申请理由 / 申请时间 / 状态 / 处理人）
   - 详情：完整理由 + 订单上下文 + 历史处理记录
   - 审批：通过 / 拒绝 + 必填备注
   - 通过后订单状态自动变 `refunded`，拒绝则保持 `paid`
3. **报表**
   - 收入 vs 成本（按模型分组）
   - 退款率趋势（30 / 90 天）

### 退款工单状态
- `pending` → `approved` / `rejected`

### 字段
- 对账记录：日期 / 渠道 / 订单数 / 总金额 / 退款数 / 退款金额 / 净额
- 退款工单：ID / 订单 ID / 用户 / 申请金额 / 申请理由 / 申请时间 / 状态 / 处理人 / 处理时间 / 备注

### 边界
- 一期 viewOrders 的"直接退款"按钮在二期改为"发起退款工单"
- 运营自己发起的退款（如客诉 + 主动补偿）走快速通道，仍需备注

## 10. M9 · 管理员账号 & 角色

### 价值
多人协作，避免所有人都能改模型价格 / 退款 / 发券。

### 用户故事
- 新员工入职 → 创建账号 → 分配"审核员"角色 → 只能看作品审核和工单
- 临时离职 → 禁用账号
- 财务负责人换人 → 切换角色

### 功能清单
1. **管理员列表**
   - 列：姓名 / 手机号 / 角色 / 状态 / 最近登录 / 创建时间
2. **新增 / 编辑**
   - 姓名 / 手机号 / 角色
   - 状态（启用 / 禁用）
3. **重置密码**（mock：toast 提示已发送临时密码）
4. **角色定义**（内置 4 个）
   - **超级管理员**：全部权限（含管理员账号管理）
   - **运营**：用户 / 项目 / 作品 / 技能 / 提示词 / 公告
   - **财务**：订单 / 对账 / 退款审批 / 套餐 / 优惠券
   - **审核员**：作品审核 / 工单反馈

### 字段
- 管理员 ID / 姓名 / 手机号 / 角色 / 状态
- 最近登录 / 创建时间 / 创建人

### 边界
- 二期角色为内置 4 个，不支持自定义角色
- 不做 SSO / 双因子认证（demo 阶段）

## 11. M10 · 操作审计日志

### 价值
敏感操作可追溯（谁、在何时、改了什么、从什么改到什么）。合规和反内鬼必备。

### 用户故事
- 出现争议（"为什么这个用户被封了"）→ 查日志定位操作人
- 月度安全审计 → 导出敏感操作记录

### 功能清单
1. **日志列表**
   - 列：时间 / 操作人 / 模块 / 操作类型 / 对象 / 摘要 / IP
   - 筛选：操作人 / 模块 / 操作类型 / 时间范围
   - 单击展开变更详情（前后值对比）
2. **导出**：CSV

### 关键操作类型枚举
- 用户：封禁 / 解封 / 赠送积分
- 项目：隔离 / 封禁 / 解除
- 作品：隐藏 / 删除
- 订单：退款（直接 / 工单审批）
- 模型：新增 / 编辑单价 / 上下架
- 技能：新增 / 编辑 / 上下架
- 提示词：新增 / 编辑 / 上下架
- 公告 / Banner：发布 / 下线
- 套餐 / 券：CRUD / 发放
- 管理员：账号增删 / 角色变更 / 密码重置

### 字段
- 日志 ID / 时间 / 操作人 ID / 操作人姓名
- 模块 / 操作类型 / 对象类型 / 对象 ID
- 变更摘要 / 变更详情（before / after JSON）
- IP / userAgent

### 边界
- 日志只读，不可删除、不可编辑
- 保留期建议 ≥ 6 个月（mock 保留全量）

## 12. M11 · 工单与反馈

### 价值
用户提的问题 / 反馈集中处理。一期前台无自助提交入口（demo 阶段），但 admin 侧需先建好结构，后续打通。

### 用户故事
- 用户反馈"视频生成卡住"→ 工单进入待处理 → 分配给审核员 → 处理 → 回复用户
- 用户追加补充 → 工单状态回到"等待处理"
- 内部备注：审核员之间留意见

### 功能清单
1. **工单列表**
   - 列：标题 / 用户 / 优先级 / 状态 / 处理人 / 创建时间 / 最近更新
   - 筛选：状态 / 优先级 / 处理人 / 时间范围
2. **详情抽屉**
   - 用户信息 + 联系方式
   - 工单描述（用户原文）
   - 处理记录时间线：运营回复 + 用户追加 + 内部备注（仅运营可见）
3. **操作**
   - 分配处理人
   - 改优先级 / 状态
   - 回复用户（mock 发送）
   - 添加内部备注
   - 关单

### 工单状态
- `new`（待分配）→ `assigned`（已分配未开始）→ `in_progress`（处理中）→ `resolved`（已解决等用户确认）→ `closed`（已关闭）
- 任意阶段可 `reopened` 回到 `in_progress`

### 字段
- 工单 ID / 标题 / 用户手机号 / 优先级（high/medium/low）
- 状态 / 处理人 / 创建时间 / 最近更新
- 描述 / 处理记录数组（含发送者类型：user/admin/system，可见性：public/internal）

## 13. 分期建议（待对齐）

11 个模块若一次性全做，工程量过大。按价值 × 工程量评估，分期参考：

| 阶段 | 模块 | 主线价值 |
|---|---|---|
| 二期 | M1 项目管理 · M5 公告 Banner · M6 充值套餐 · M9 管理员账号 | 治理基础 + 营收链路打通 |
| 三期 | M2 作品审核 · M3 技能库 · M7 优惠券 · M8 退款审批 | 内容生态 + 营销 |
| 四期 | M4 提示词库 · M10 操作日志 · M11 工单 | 精细化 + 体验补强 |

> 这是默认建议，具体节奏和优先级看产品 / 运营当前最缺什么，待你拍板。

## 14. 通用边界 / 非范围
